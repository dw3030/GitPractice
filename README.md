| Git Command                                                  | What it does                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| git                                                          | shows common git commands and usage                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| git config --global user.name "FirstName LastName"           | configure your name with Git                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| git config --global user.email "email@address.com"           | configure your email with Git                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| git init                                                     | create a new repo in whatever dir we are in (USE GIT STATUS TO ENSURE YOU'RE NOT ALREADY IN A REPO)<br>creates a .git hidden folder which contains the repo (if you delete .git, it will delete the entire git history)                                                                                                                                                                                                                                                                                       |
| cat .git/HEAD                                                | displays a ref of where the HEAD is pointing to                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| git status                                                   | inspects contents of the working directory and staging area                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| git add <filename>                                           | adds file from the working directory to staging area                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git add <filename1> <filename2> etc…                         | adds files from the working directory to staging area                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| git add .                                                    | adds all files from working dir to staging area                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| git commit                                                   | permanently stores changes from the staging area to inside the repo                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| git commit -m "message"                                      | git message conventions: must be in quotes, written in present tense, should be 50 char or less                                                                                                                                                                                                                                                                                                                                                                                                               |
| git commit -a -m "message"                                   | adds all unstaged changes before committing with a message                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| git commit --amend                                           | this will overwrite your previous commit with any changes added to the staging area…you will be asked to update your commit message                                                                                                                                                                                                                                                                                                                                                                           |
| git commit --amend --no-edit                                 | this is if you choose to keep the previous commit's message                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| git push                                                     | pushes and saves commited changes permanently to the repo                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git diff                                                     | shows all changes in the working dir not staged for the next commit                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| git diff <filename>                                          | shows differences in the file between working directory and staging area versions                                                                                                                                                                                                                                                                                                                                                                                                                             |
| git diff HEAD                                                | shows all changes (staged and unstaged) in the working dir since the last commit (diff btw HEAD and working dir)                                                                                                                                                                                                                                                                                                                                                                                              |
| git diff HEAD <filename>                                     | shows all changes (staged and unstaged) in the working dir since the last commit but just for only the specified <filename>                                                                                                                                                                                                                                                                                                                                                                                   |
| git diff HEAD <commitHash>                                   | shows changes btw current HEAD and previous commit specified by hash                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git diff --staged or git diff --cached                       | shows only the staged changes (either works)                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| git diff --staged <filename> or git diff --cached <filename> | shows only the staged changes for a particular filename                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| git diff branch1..branch2, or git diff branch1 branch2       | shows the changes between the tips of the 2 indicated branches                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| git diff branch1..branch2 <filename>                         | shows the changes between the tips of the 2 indicated branches but only for specified filename                                                                                                                                                                                                                                                                                                                                                                                                                |
| git log                                                      | shows commits stored chronologically in the repo,<br>lists the commit author,<br>lists date/time of commit,<br>lists commit message                                                                                                                                                                                                                                                                                                                                                                           |
| git log --oneline                                            | summarized git log in oneline format                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git log -S "keyword"                                         | displays a list of commits that contain the keyword in the commit message (this is why it's useful to make distinct commit messages, for retrieval later),<br>the keyword is in quotes                                                                                                                                                                                                                                                                                                                        |
| git log --oneline --graph                                    | displays a visual representation of how the branches/commits were created                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git log --graph --decorate --oneline --all                   | displays a visual representation decorated with color for ease of reading                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git show HEAD                                                | shows the HEAD commit, which in most cases is the one you are currently on,<br>this displays everything the git log command displays for the HEAD commit, plus all the file changes that were committed                                                                                                                                                                                                                                                                                                       |
| git checkout HEAD <branchName>                               | restores the file in your working dir to look exactly as it did when you last made a commit                                                                                                                                                                                                                                                                                                                                                                                                                   |
| git checkout HEAD .                                          | does this for all files                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| git checkout HEAD~1                                          | checks out the commit right before HEAD…if you want to go 2 or 3 commits back, you can use 'git checkout HEAD~2' or 'git checkout HEAD~3'                                                                                                                                                                                                                                                                                                                                                                     |
| git checkout HEAD <fileName>, or git checkout -- <fileName>  | discards any changes in the specified file, reverting it back to what it was at HEAD                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git checkout -- <branchName>                                 | does the same thing as 'git checkout HEAD <filename>                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git checkout -b <branchName>                                 | checks out a branch that you create on the same line                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| git checkout <oldCommitHash>                                 | checks out an older commit and detaches from the HEAD                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| git reset HEAD <filename>                                    | resets the file in the staging area to be the same as it was in the HEAD commit…it does not discard the file changes from the working dir, it just removes them from staging,<br>(it returns M (short for 'modification') next to the file name that was reset)                                                                                                                                                                                                                                               |
| git reset <commit\_hash>                                     | resets to the commit specified by the first-7-digits-of-the-SHA (the hash),<br>the commits following where you reset to are gone and the HEAD is moved to the commit you reset to<br>BUT…the changes are still in the working directory                                                                                                                                                                                                                                                                       |
| git reset --hard <commit\_hash>                              | resets to the commit specified, but also removes the changes from the working directory                                                                                                                                                                                                                                                                                                                                                                                                                       |
| git reset --hard <commit\_hash> HEAD~1                       | resets to the commit right before the HEAD, so it basically removes the last commit and all changes in it from the working directory                                                                                                                                                                                                                                                                                                                                                                          |
| git stash                                                    | allows you to get back to a clean commit point with a sychronized working tree, and avoid losing local changes in the process,<br>you are stashing local work temporarily in order to update a previous commit and later on retrieve your work                                                                                                                                                                                                                                                                |
| git config --global alias.<shorterCommand> "defaultCommand"  | with git config and alias, you can change git commands to whatever you want to shorten them,<br>for example, if you want to change 'checkout' to 'co', you'd type:<br>git config --global alias.co "checkout"                                                                                                                                                                                                                                                                                                 |
| git remote                                                   | creates the path to the remote repo (gives it name of origin by default, if it was from a cloned repo)                                                                                                                                                                                                                                                                                                                                                                                                        |
| git remote -v                                                | shows the path for the remote repo (-v means 'verbose'), useful to show you that the remote repo was created successfully                                                                                                                                                                                                                                                                                                                                                                                     |
| git remote add <origin> <remoteRepoUrlPath>                  | connects the local repo (usually named 'origin') to the destination of the remote repo (identifed by the github repo url), adds the remote path to the local repository                                                                                                                                                                                                                                                                                                                                       |
| git branch                                                   | this will tell you what branch you are on                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git branch -v                                                | this will tell you what branch you are on, but with more info including the most recent commit hash and commit message                                                                                                                                                                                                                                                                                                                                                                                        |
| git branch <newBranchName>                                   | this will create a new branch…branch names can't contain whitespaces…to move to the new branch, use git checkout...<br>Best practice for naming a branch is branchTicketNumber\_branchAuthor\_branchType\_shortBranchDescription                                                                                                                                                                                                                                                                              |
| git branch -d <branchName>                                   | this will delete the specified branch from the project<br>you can't delete a branch that you're currently on                                                                                                                                                                                                                                                                                                                                                                                                  |
| git branch -D <branchName>                                   | this will delete the specified branch from the project IF IT HAS NOT BEEN FULLY MERGED                                                                                                                                                                                                                                                                                                                                                                                                                        |
| git branch -m <newBranchName>                                | you have to be on the branch to rename it<br>renames the branch                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| git branch -r                                                | shows the remote-tracking branches our local repo knows about                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| git merge origin/main                                        | this will merge the changes fetched from the remote repo/branch (origin/main) into the local repo                                                                                                                                                                                                                                                                                                                                                                                                             |
| git merge <branchName>                                       | merge the named branch into the branch you are using (ie. Use git checkout main, then git merge the other branch into it)                                                                                                                                                                                                                                                                                                                                                                                     |
| git clone <remoteLocation> <cloneName>                       | clones a repo from remoteLocation (web address, filepath, etc) to the directory cloneName,<br>changes made to the clone on the local machine will not affect the repo that it was cloned from                                                                                                                                                                                                                                                                                                                 |
| git fetch                                                    | brings all changes from the remote repo down onto a 'remote branch',<br>this does NOT merge the changes from the remote into your local repo                                                                                                                                                                                                                                                                                                                                                                  |
| git fetch <remote>                                           | fetches branches and history from a specified remote repo…it only updates the remote tracking branches<br>if you want to integrate the fetched changes, you need to checkout the branch and merge them into the local repo                                                                                                                                                                                                                                                                                    |
| git fetch <remote> <branch>                                  | if you only want to fetch one of the remote repo's branches                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| git push origin <branchName>                                 | this will push branchName up to the remote repo (origin)… the owner of the remote repo will then be able to review the changes and merge them into the main branch of the remote repo                                                                                                                                                                                                                                                                                                                         |
| git push -u origin main                                      | the pattern is 'git push <remote> <branch>' … so in this case, we are saying push the changes to the upstream (the '-u') remote 'origin' on the branch 'main' …this saves the changes to the remote repo on the main branch … if we set the upstream branch with the -u flag, then we can use just git push when pushing up to the remote repo                                                                                                                                                                |
| git push origin <localBranchName>:<diffRemoteBranchName>     | if we want to push a local branch to a remote branch of a different name, use this format                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git rebase <branchNameYouWantRebasedTo>                      | git rebase will take the latest changes from the rebaseTo branch (usually main) and incorporate them into the base of the checkedout branch you're working on<br>reapplies commits from one branch to another base branch…rewrites commit history…useful whenever we want to add changes of a base branch back to a branched out branch,<br>common rule is to use rebase only locally, and to not rebase after a push to remote because it convolutes the commit history<br>also don't rebase a shared branch |
| git rebase -i HEAD~4                                         | the -i flag enters the interactive mode, which allows us to edit commits, add files, drop commits, etc…note, we need to specify how far back (here indicated by HEAD~howFarBack) we want to rewrite the commits…<br>Note also, we are not rebasing onto another branch here, we are only rebasing a series of commits onto the HEAD they're currently based on                                                                                                                                                |
| git remote add upstream <httpsPathForForkedProjectRepo>      | sets the upstream to the parent repo...setting the upstream of a cloned project allows us to get updates from the project into our forked repo                                                                                                                                                                                                                                                                                                                                                                |
| git fetch upstream                                           | once the upstream is set to the parent repo, we use this command to make our forked repo up to date with the parent repo                                                                                                                                                                                                                                                                                                                                                                                      |
| git pull                                                     | this will download all the changes from the point where the local and main diverged,<br>A pull request is a discussion page for a set of code changes btw one branch and another                                                                                                                                                                                                                                                                                                                              |
| git pull <remoteName> <branchName>                           | this pulls a specified remote & branch to where we are and merges them into the branch we run this command from                                                                                                                                                                                                                                                                                                                                                                                               |
| git pull origin main                                         | this fetches the latest info from the origin's main branch and merges those changes into our current branch                                                                                                                                                                                                                                                                                                                                                                                                   |
| git switch <branchName>                                      | switch to branch indicated                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| git switch -c <branchName>                                   | switch to a branch that you're creating on the same line                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| git switch -                                                 | switch the branch you were last on                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| git stash, or git stash save                                 | saves changes that are not yet committed…you can stash changes and come back to them later<br>it takes all uncommitted changes (staged and unstaged) and stashes them, reverting the changes in your working copy                                                                                                                                                                                                                                                                                             |
| git stash pop                                                | retrieves the work that was stashed…<br>you can stash changes, then move to prior commits (for example, on other files), commit THOSE changes, then use git stash pop to retrieve the changes you made that were stashed,<br>you don't necessarily need to be on the same branch as when you initially made the changes to use 'git stash pop' to retrieve                                                                                                                                                    |
| git stash apply                                              | applies the stash changes WITHOUT removing them from the stash…this is useful if you want to apply the stashed changes to multiple branches                                                                                                                                                                                                                                                                                                                                                                   |
| git stash apply stash@{numberOfStashFromStashList}           | if you have multiple stashed changes, this will apply only the specified stash changes indicated by where they are in the stash list                                                                                                                                                                                                                                                                                                                                                                          |
| git stash list                                               | view what is in the stash                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| git stash drop stash@{numberOfStashFromStashList}            | deletes a particular stash                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| git stash clear                                              | completely drops the entire stash                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| git restore <fileName>                                       | reverts the fileName to back to what it looked like at HEAD (same as git checkout HEAD <filename>)                                                                                                                                                                                                                                                                                                                                                                                                            |
| git restore --source HEAD~1 <filename>                       | reverts the fileName to back to what it looked like at one commit (specify another number if necessary) before HEAD                                                                                                                                                                                                                                                                                                                                                                                           |
| git restore --staged <filename>                              | this will remove a specified file from the staging area that you had already added                                                                                                                                                                                                                                                                                                                                                                                                                            |
| git revert <commitHash>                                      | this will create a new commit which reverses the changes from a commit…because it results in a new commit, you will be prompted to enter a commit message,<br>the commit you're reverting will still show up in git log, but the changes from it are gone                                                                                                                                                                                                                                                     |
| git remote                                                   | shows you what remote name is associated with the current repo                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| git tag                                                      | prints all the tags in the current repo                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| git tag -l '\*beta\*'                                        | search for tags that match a particular pattern…in this example, the asterisks are wildcards and this command will print a list of tags with 'beta' in their name                                                                                                                                                                                                                                                                                                                                             |
| git checkout <tag>                                           | a tag refers to a commit and not a branch, but you CAN checkout a tag like you would checkout a commitHash (puts you in detached HEAD)                                                                                                                                                                                                                                                                                                                                                                        |
| git tag <tagname>                                            | creates a lightweight tag, by default Git will reference the commit that HEAD is referencing                                                                                                                                                                                                                                                                                                                                                                                                                  |
| git tag -a <tagname>                                         | creates an annotated tag…git will open the text editor for more info to be added to the tag<br>you can also use -m like a commit to just annotate the tag in the terminal (git tag -a -m <tagname> "note")                                                                                                                                                                                                                                                                                                    |
| git show <tagname>                                           | will show the annotated metadata for a tag indicated                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| gti tag <tagname> <commit>                                   | will tag an older commit with the commit hash (can also annotate it: git tag -a <tagname> <commitHash>)                                                                                                                                                                                                                                                                                                                                                                                                       |
| git tag -d <tagname>                                         | deletes a particular tag                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| git push --tags                                              | pushing up to a remote repo doesn't contain tags by default, so you need to push tags up with this command in addition to the commits pushed up …to only push up a single tag, use 'git push <remoteName> <tagName>                                                                                                                                                                                                                                                                                           |
| git reflog show HEAD                                         | will show the log of HEAD reference…if you want to see the logs for the tip of a branch, use git reflog show <branchName>                                                                                                                                                                                                                                                                                                                                                                                     |
